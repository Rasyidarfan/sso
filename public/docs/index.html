Dokumentasi SSO BPS

Panduan integrasi sistem Single Sign-On BPS dengan aplikasi internal

Pendahuluan

SSO BPS menggunakan mekanisme redirect untuk autentikasi. Setelah berhasil login, pengguna akan diarahkan kembali ke aplikasi dengan membawa authorization code.

Base URL: sso.bps9702.com/v1/
Penting: Sebelum menggunakan API, pastikan aplikasi Anda sudah terdaftar dan mendapatkan:
  • client_id
  • client_secret
  • redirect_uri (URL yang sudah didaftarkan)

Alur Autentikasi

  1. Aplikasi mengarahkan pengguna ke halaman login SSO menggunakan client_id yang terdaftar
  2. Pengguna login di halaman SSO
  3. SSO mengarahkan pengguna kembali ke redirect_uri yang sudah terdaftar dengan authorization code
  4. Aplikasi menukar authorization code dengan data pengguna

Endpoints

1. Login Redirect

GET sso.bps9702.com/v1/authorize

Endpoint untuk mengarahkan pengguna ke halaman login SSO.

Query Parameters:

Parameter Tipe Keterangan
client_id string ID aplikasi yang sudah terdaftar required
state string Random string untuk keamanan optional

Contoh Request:

GET sso.bps9702.com/v1/authorize?client_id=app123&state=xyz123

Success Response:

Redirect ke URI callback yang telah didaftarkan dengan code authorization dan parameter state (jika disediakan)

HTTP/1.1 302 Found Location: https://example.com/callback?code=abc123&state=xyz123

Error Codes:

Status Code Error Code Deskripsi
400 MISSING_PARAMETERS Parameter client_id tidak ditemukan
401 UNAUTHORIZED Pengguna belum login
400 CLIENT_ID_NOT_FOUND Client ID tidak tersedia
401 INVALID_CLIENT Client ID tidak valid atau aplikasi tidak aktif
405 METHOD_NOT_ALLOWED Metode HTTP yang digunakan tidak benar (harus GET)

2. Exchange Code

POST sso.bps9702.com/v1/token

Endpoint untuk menukar authorization code dengan data pengguna.

Request Body:

Parameter Tipe Keterangan
code string Authorization code dari redirect required
client_id string ID aplikasi required
client_secret string Secret key aplikasi required

Success Response:

{ "status": "success", "data": { "user_id": "340055640", "name": "John Doe", "nip_9": "340055640", "nip_18": "198507212011012018", "email": "john.doe@bps.go.id", "roles": ["admin", "ipds"] } }

Error Codes:

Status Code Error Code Deskripsi
400 INVALID_REQUEST Parameter tidak lengkap atau tidak valid
401 INVALID_CLIENT Client ID tidak valid atau aplikasi tidak aktif
401 INVALID_CLIENT_SECRET Client Secret tidak valid
400 INVALID_GRANT Authorization code tidak ditemukan, sudah digunakan, atau sudah expired
403 INACTIVE_USER User tidak aktif
405 METHOD_NOT_ALLOWED Metode HTTP yang digunakan tidak benar (harus POST)

Contoh Implementasi

1. Redirect ke SSO:

// PHP $client_id = "your_client_id"; // ID yang didapat saat pendaftaran $state = bin2hex(random_bytes(16)); $_SESSION['oauth_state'] = $state; $url = "sso.bps9702.com/v1/authorize?" . http_build_query([ 'client_id' => $client_id, 'state' => $state ]); header("Location: " . $url);

2. Callback Handler:

// PHP if (!isset($_GET['state']) || $_GET['state'] !== $_SESSION['oauth_state']) { die('Invalid state parameter'); } if (!isset($_GET['code'])) { die('Authorization code not provided'); } $code = $_GET['code']; $ch = curl_init(); curl_setopt($ch, CURLOPT_URL, "sso.bps9702.com/v1/token"); curl_setopt($ch, CURLOPT_POST, 1); curl_setopt($ch, CURLOPT_POSTFIELDS, http_build_query([ 'code' => $code, 'client_id' => 'your_client_id', 'client_secret' => 'your_client_secret' ])); curl_setopt($ch, CURLOPT_RETURNTRANSFER, true); $response = curl_exec($ch); curl_close($ch); $result = json_decode($response, true); if (isset($result['status']) && $result['status'] === 'success') { // Authentication successful $_SESSION['user'] = $result['data']; header("Location: /dashboard.php"); } else { // Authentication failed echo "Error: " . ($result['message'] ?? 'Unknown error'); }

Penanganan Error

API ini menggunakan format error yang konsisten. Setiap respon error akan memiliki format berikut:

{ "status": "error", "message": "Pesan error yang menjelaskan masalah", "error_code": "KODE_ERROR", "errors": { ... } // detail validasi jika ada }

Selalu periksa status code HTTP dan kode error untuk menangani error dengan tepat.

HTTP Status Jenis Error Keterangan
400 Bad Request Parameter tidak lengkap atau tidak valid
401 Unauthorized Client ID/secret tidak valid atau user belum login
403 Forbidden User tidak memiliki akses atau tidak aktif
405 Method Not Allowed Metode HTTP yang digunakan tidak benar
